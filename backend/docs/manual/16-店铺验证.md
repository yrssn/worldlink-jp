# 店铺验证

一、模块整体说明

1.1 上下游来源规则

本模块归属：账号部→店铺校验管理，仅接收【店铺 / 支付】注册类型单据，银行类型单据不走本模块；单据两种来源：

无需银行资料（订单池未勾选需要银行）：注册表审核通过 → 直接推送至店铺校验管理；

需要银行资料（订单池勾选需要银行）：银行验证管理审核通过 → 推送至店铺校验管理；

注册类型 = 银行：银行核验通过直接进入【银行卡管理】，永远不流入店铺校验。

1.2 下游分流规则（核心）

店铺校验审核时，弹窗字段【是否成为库存】为必选项：

✅ 选择「是」：审核通过后，单据自动流转至【账号部→账号管理（库存模块）】，生成库存账号档案，注册表回填库存编号；

❌ 选择「否」：审核完结，终止全流程，不进入账号管理、不生成库存数据。

1.3 使用角色

店铺校验审核专员、平台注册管理员、超级 Admin。

二、页面功能详情（参照原型页面）

2.1 顶部筛选区

查询条件：项目编号、平台（下拉模糊）、法人、店铺种类、单据状态；

功能按钮：搜索、重置（清空所有查询条件）。

2.2 列表字段

申请人、申请时间、平台、属地、属性、法人、公司、店铺种类（固定：店铺 / 支付）；左下角分页：每页条数切换、总数据量、页码跳转。

菜单右上角红点：代表待审核单据数量。

2.3 右侧弹窗【店铺校验】详情页

1）只读回显字段（上游同步，不可编辑）

平台、法人（自动带出注册表 / 银行验证上游数据）

录入字段（店铺审核员手动填写）

三、业务审核流转规则

3.1 审核通过

弹窗【是否成为库存 = 是】：单据办结，数据自动推送 ** 账号管理（库存）** 模块入库，注册表原单据回填库存编号，注册表状态变更【完成】；

弹窗【是否成为库存 = 否】：单据直接归档完结，不推送账号管理，无库存数据生成，流程结束。

3.2 审核驳回

填写驳回备注，单据原路退回：退回注册表【审核中】，注册经办人重新修改资料；
