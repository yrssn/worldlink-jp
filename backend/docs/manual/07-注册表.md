# 注册表

## 文档概述

### 项目背景

本模块属于进销存系统 - 平台注册部子功能，数据源头来自【订单池】，订单池完成资料校验无误后，单据流转生成注册表待处理数据；由平台注册部管理员分配任务、经办人资料填报、多级审核后，最终流转至银行验证 / 店铺验证，全流程终审通过后数据落库至【账号部 - 库存（账号管理模块）】。

### 模块定位

归属左侧菜单栏「平台注册部→注册表」，关联上游：订单池；下游：银行验证、店铺验证、账号库存模块；操作人员：平台注册部管理员、注册经办人。

### 目的

规范主体注册资料全生命周期管理，实现任务指派、附件上传、多级审核、节点分流、数据归档入库全流程线上管控。

## 整体业务流程图

订单池（资料校验无误）→自动生成注册表单据 →平台注册部管理员【指派】→经办人接收任务上传底表附件 →管理员审核底表

审核不通过：退回经办人重新修改上传附件；

审核通过：需银行资料：流转至【银行验证】节点；银行审核通过→店铺验证；

无需银行资料：跳过银行验证，直接流转【店铺验证】；

店铺验证审核通过 →单据完成，自动同步数据至【账号部→账号管理（库存）】模块，paypal/stripe【项目一部→账号管理（库存）】模块

## 状态标签栏

全部：汇总所有注册表单据总数；

待指派：订单池生成单据，尚未被管理员指派给注册员工；

进行中：已指派，经办人接收任务，待上传底表资料；

审核中：经办人已上传底表，等待管理员资料审核；

银行验证：底表审核通过，需银行资料，进入银行核验节点；

店铺验证中：银行核验通过 / 免银行直接跳转，等待店铺资料核验；

完成：全流程审核结束，数据入库账号库存模块；

失败：全流程任意节点驳回标记失败，流转回订单提交池；

取消：作废终止流程，流转回订单提交池；

暂停：人工暂停；
