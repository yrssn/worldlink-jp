# 负责人待办

本模块是订单分配与法人主体流转的核心环节，承接「订单池」已接单的订单，由日联部负责人完成法人主体分配，再流转至对应「担当」或「银行卡使用申请」的待办事项，驱动后续法人资料准备流程。

1. 接收订单并分配法人主体

进入「负责人 - 待办事项」，筛选状态为 **「待分配」** 的订单（

点击订单操作列的【分配】按钮，进入分配页面，根据订单的平台、属地、企业国别等业务属性，匹配并指派对应法人主体：

系统会展示可分配的法人主体列表（需与订单业务要求匹配）

选择法人主体后，系统自动关联该主体对应的「担当」人员

确认分配信息无误后，提交分配操作

2. 订单流转至担当待办

分配提交后，订单状态由「待分配」更新为 **「担当处理中」**

系统自动在该法人主体对应的「担当 - 待办事项」模块中，生成一条新的待处理任务，如果资料含有银行的话，会流转至银行卡使用申请模块，账号部负责人审核通过再流转至担当待办

担当人员登录系统后，即可看到该订单任务，开始法人资料准备工作

3. 批量分配 / 审核 / 拒绝

批量分配：勾选多条「待分配」订单，批量指派同一类法人主体（如日本企业法人），适合同一业务线订单集中处理（需保证订单编号一致）

批量审核：对「资料审核中」的订单，批量通过 / 驳回审核

批量拒绝：对不符合业务要求的订单，批量驳回并填写原因，订单状态更新为「已拒绝」并流转回订单池由平台注册部管理员处理（需保证订单编号一致）
