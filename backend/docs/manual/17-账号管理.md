# 账号管理

模块概述

数据来源

归属：账号部→账号管理，仅接收【店铺校验管理】审核结果为【是 - 成为库存】的单据；

上游链路：订单池→注册表→（按需走银行验证）→店铺校验→勾选「是成为库存」→审核通过后自动生成本条库存数据、生成唯一库存编号，同步回填注册表库存编号字段；

店铺校验选「否 - 不成为库存」、注册类型 = 银行（入银行卡管理）均不会流入本模块。

业务用途

本模块是店铺账号库存台账，存储全量可对外售卖的注册店铺主体资源，用于后续销售出库、对接客户档案；库存状态管控（可售 / 已售 / 已封号），统一管理店铺账号、邮箱、收款等核心资料。

页面功能详情

顶部批量功能

批量封号：勾选多条库存数据，批量变更账号状态为【已封号】。

列表展示字段

库存编号、项目号、拿店日期、平台、属性、属地、法人姓名、公司名字、账号状态、操作按钮。

账号状态枚举：可售 / 已售 / 已封号（原型可见：可售、已售）。

单行操作按钮

编辑（蓝色笔形按钮） 打开编辑弹窗，修改店铺基础资料：平台账号 ID / 账号密码、邮箱账号 / 邮箱密码、店铺链接、收款账户、备注等底层信息；

封号（红色按钮） 单条手动封号，账号状态由【可售】变更为已封号，封号后无法再对外销售出库；

查看资料（绿色按钮） 跳转 / 弹窗查看客户档案关联信息（法人、主体备案、注册底表、历史对接客户记录），读取全链路上游注册表 + 银行 + 店铺校验归档附件。

核心业务规则

库存单据生成规则

触发条件：店铺校验审核通过 + 是否成为库存 = 是；

系统自动生成：唯一【库存编号】、拿店日期（默认店铺审核通过当日），自动携带上游全量字段：项目号、平台、属地、属性、法人、公司名、银行收款卡号等；

反向回写：生成库存编号后，自动回写到源注册表单据的库存编号字段，注册表状态变更为【完成】。

账号状态流转规则

可售：资源未卖给客户，可对外售卖

出售中：正在走流程卖出给客户

已售：资源已卖给客户，库存占用，不再对外售卖

已封号：库存管理员自行操作

编辑规则

仅可修改店铺账号、邮箱、收款、备注等业务资料；库存编号、项目号、法人、属地等源头基础字段不可修改。

查看资料规则

按钮关联客户档案模块，自动关联该法人 / 主体全生命周期资料：上游注册底表附件、银行开户资料、店铺注册资料、历史成交客户信息。
