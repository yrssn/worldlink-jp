# 银行卡使用申请管理

模块整体说明

模块归属与单据来源

归属：账号部→银行卡使用申请管理，左侧菜单角标数字代表待审核单据总量；

上游来源：订单池单据勾选【需要银行卡】，日联部完成法人主体分配后自动生成本条银行卡使用申请单据，流入本模块待审核列表；订单池未勾选需要银行卡的单据不生成本模块申请。

业务全链路前置 & 后置流向

订单池勾选需要银行卡→日联部分配法人主体→生成【银行卡使用申请】→账号部管理员审核

审核可用：分配银行卡：单据流转至日联部担当，由担当继续审核法人其余资料；

审核不可用：单据退回日联部负责人待办列表，负责人更换法人主体后重新提交银行卡申请；

审核无需使用：跳过银行卡审核阶段，不选择银行卡的情况下流转至担当待办

模块作用

管控订单主体的银行卡资源分配：可从现有库存银行卡挑选分配给对应法人、可现场新增法人名下银行卡并绑定使用，也可无卡直接放行流转。

页面列表功能

顶部筛选区

筛选字段：订单编号、平台、企业国别（模糊检索）；按钮：搜索、重置。

状态标签页

全部 ：汇总全部申请单据；

待处理 ：账号部待审核单据；

已处理 ：已完成审核（通过 / 驳回）的历史单据。
