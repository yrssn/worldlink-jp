# 银行卡管理

## 模块整体概述

### 数据来源与业务定位

模块归属：账号部→银行卡管理

自动来源：银行验证管理【注册类型 = 银行】单据

银行验证审核通过后，系统自动新增一条银行卡档案至本页面，不进入店铺校验、不进入账号管理（库存）；

手动来源：页面【添加】按钮人工新增银行卡信息；银行卡使用申请管理模块中手动新增

下游联动：新增后同步推送至财务系统银行卡模块，财务自动生成对应银行卡台账数据；

### 核心约束

银行卡号全局唯一：手动新增、系统自动同步时，卡号重复则拦截保存并弹窗提示。

### 业务用途

统一存储全量独立开户类银行卡主体资料，作为平台收款卡资源台账，对接财务对账、付款审核模块。
