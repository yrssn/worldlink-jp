# 库存审核管理

## 模块概述

### 来源与业务定位

归属：账号部→库存审核管理

单据来源：销售发起给店申请，系统自动区分库存属地：

非欧美地区库存：生成单据流入本【库存审核管理】待审；

欧美地区库存：不进入本模块，进入项目一部库存审核。

业务目的：审核出库店铺账号资料完整性，校验账号 ID 必填项，管控非欧美库存出库准入，是销售出库前置审核节点。

### 上下游流转规则

审核通过：单据流转【给店审核管理】继续下一环节审批；

审核驳回（失败）：单据原路退回销售给店申请列表，单据状态标记「审核失败」，销售可重新编辑提交。

### 核心校验约束

审核操作强制校验：账号 ID 不能为空，账号 ID 空时禁止审核通过，弹窗提示补全账号信息。

## 页面功能详情

### 顶部筛选查询区

检索字段：库存编号、给店编号、客户姓名、平台、申请人、申请时间（起止日期）；

操作按钮：搜索、重置。

### 顶部批量操作

批量通过：批量勾选多条待处理单据，批量审核通过（批量同样强制每条单据账号 ID 非空校验，任意一条账号 ID 为空则整批无法批量通过）。

### 状态标签分类

待处理：待账号管理员审核的销售出库申请；

已同意：审核通过，已推送至给店审核；

已拒绝：审核驳回，退回销售给店列表。
