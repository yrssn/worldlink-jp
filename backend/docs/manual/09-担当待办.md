# 担当待办

资料类任务处理（确认资料类型）

进入「担当 - 待办事项」，筛选类型为「确认资料」、状态为「待处理」的任务

点击【资料处理】按钮，进入法人主体资料详情页

核对 / 补充法人主体资料（如公司信息、注册资料、合规文件等）

确认资料无误后，提交审核通过

系统自动将审核结果反馈至「负责人 - 待办事项」，任务状态更新为「已完成」

若资料有误，点击【拒绝】按钮，填写拒绝原因，任务状态更新为「已拒绝」，流转回负责人待办重新处理

付款类任务处理（付款类型）

接收任务时，订单已进入库存，系统自动生成库存编号（作为付款流程的起点标识）

点击【付款处理】按钮，进入付款申请详情页，填充付款信息（金额、账户、用途等），有中间人则会在「付款申请管理」额外生成一条介绍金类型数据

审核通过后，系统自动在「付款申请管理」模块生成对应付款数据

任务状态更新为「已完成」，付款流程进入下一环节

转审操作（同部门流转）

对于「付款」类型的任务，担当可点击【转审】按钮，将任务转给同部门的其他同事

转审时需填写转审备注，说明转审原因

转审后，后续付款流程中，所有相关节点（如付款申请管理）都会显示转审后的员工担当姓名，而非原担当

转审仅支持同部门内流转

协助处理（协助类型）

注册部员工在「注册表」模块，发现法人主体资料缺失 / 不完整

点击「待办事项」按钮，选择对应担当人员，填写需补充的资料项（如法人名称、注册文件、合规资料等）

提交请求后，系统在「担当 - 待办事项」生成一条「协助」类型的待处理任务

进入「担当 - 待办事项」，筛选类型为「协助」、状态为「待处理」的任务

点击【协助处理】按钮，进入资料处理详情页

按请求要求，上传缺失的法人主体资料、填写补充说明

确认资料无误后，提交处理结果

系统自动将处理结果（含上传的资料、反馈说明）反馈给发起请求的注册表员工

任务状态更新为「已完成」，资料补全流程结束
