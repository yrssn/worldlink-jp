# 付款申请处理

模块概述

本模块为进销存系统 - 日联部下属核心功能，用于集中管理全平台付款申请单的全生命周期流程，覆盖付款申请的生成、查询、审批流转与状态管理。所有数据均来源于「担当待办事项」，按业务规则自动生成不同付款类型的申请单，支持多维度筛选、状态追踪与审批流程可视化，确保付款业务的合规性与可追溯性。

拿店：客户 / 项目入驻电商平台时产生的基础业务款项，含入驻费、保证金、平台服务费等由「担当待办事项」中的付款类业务流程触发，为基础业务款，与介绍人信息无关

介绍金：支付给客户引荐方（中间人）的佣金款项，属于附加业务费用	需同时满足两个条件：

① 该付款对应的法人主体，在「客户档案」中已维护完整的介绍人 / 中间人信息；

② 担当在处理对应待办事项时，明确勾选了「需要收取介绍金」选项

注：同一法人主体可同时存在「拿店」和「介绍金」两类付款申请，两类单据独立生成、独立审批。

审批流程与状态管理

审批流程入口：每笔付款申请对应「审批流程」操作按钮，点击可查看该单据的完整审核链路，包含当前节点、审批人、流转时间、审批意见等信息。

状态字段说明：

已完成：所有审批节点全部通过，付款流程闭环，款项已按流程处理。

审核中：已提交审批，当前处于某一审批节点待处理，流程尚未结束。

待审核：尚未进入审批流程。

审核失败：中途被审批节点打回，需修改后重新提交。

状态联动规则：审批流程的节点进度与「状态」字段自动同步更新，审批节点完成后状态实时变更，确保数据一致性。
